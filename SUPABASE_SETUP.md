# Инструкция по настройке нового проекта Supabase

## Шаг 1: Получение API ключей

1. В вашем новом проекте Supabase перейдите в **Settings** (настройки) → **API**
2. Найдите следующие значения:
   - **Project URL** - это ваш `NEXT_PUBLIC_SUPABASE_URL`
   - **anon public** ключ - это ваш `NEXT_PUBLIC_SUPABASE_ANON_KEY`
3. Скопируйте оба значения (они понадобятся на шаге 3)

## Шаг 2: Создание таблиц в базе данных

1. В дашборде Supabase перейдите в **SQL Editor** (SQL редактор)
2. Нажмите **New query** (Новый запрос)
3. Откройте файл `supabase_setup.sql` из корня проекта
4. Скопируйте весь SQL код из файла
5. Вставьте код в SQL Editor
6. Нажмите **Run** (или `Ctrl/Cmd + Enter`)
7. Должно появиться сообщение об успешном выполнении

## Шаг 3: Обновление переменных окружения на Vercel

1. Зайдите на [vercel.com](https://vercel.com) и откройте ваш проект
2. Перейдите в **Settings** → **Environment Variables**
3. Найдите и обновите следующие переменные:
   - `NEXT_PUBLIC_SUPABASE_URL` - вставьте Project URL из шага 1
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY` - вставьте anon public ключ из шага 1
4. Если переменных нет - создайте их (Add New)
5. После обновления нажмите **Save**

## Шаг 4: Пересборка проекта на Vercel

1. После обновления переменных окружения Vercel автоматически пересоберет проект
2. Или вы можете вручную запустить пересборку: **Deployments** → выберите последний деплой → **Redeploy**

## Шаг 5: Проверка

1. После деплоя откройте ваш сайт
2. Попробуйте зарегистрировать нового пользователя
3. Попробуйте войти в аккаунт
4. Если все работает - отлично! ✅

## Важно!

- После создания нового проекта все пользователи должны зарегистрироваться заново
- Все тесты и результаты будут потеряны (если не восстанавливали из бэкапа)
- Убедитесь, что в настройках Supabase включена аутентификация по email

## Если что-то не работает

1. Проверьте консоль браузера на наличие ошибок
2. Проверьте, что переменные окружения правильно установлены на Vercel
3. Убедитесь, что SQL скрипт выполнен без ошибок
4. Проверьте логи на Vercel в разделе **Deployments**
