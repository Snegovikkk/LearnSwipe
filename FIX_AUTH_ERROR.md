# Исправление ошибки "AuthRetryableFetchError"

## Проблема
Ошибка "Load failed" с кодом "AuthRetryableFetchError" означает, что приложение не может подключиться к Supabase.

## Причина
В диагностике видно, что используется новый проект Supabase (`mywoevhpkxjheyikqliz.supabase.co`), но переменные окружения на Vercel могут быть устаревшими или неправильными.

## Решение

### Шаг 1: Получите правильные ключи из нового проекта Supabase

1. Зайдите в ваш новый проект Supabase
2. Перейдите в **Settings** → **API**
3. Скопируйте:
   - **Project URL** (например: `https://mywoevhpkxjheyikqliz.supabase.co`)
   - **anon public** ключ (должен начинаться с `eyJhbGciOiJIUzI1NiIs...` - это JWT токен)

### Шаг 2: Обновите переменные окружения на Vercel

1. Зайдите на [vercel.com](https://vercel.com) → ваш проект
2. Перейдите в **Settings** → **Environment Variables**
3. Найдите и обновите следующие переменные:

   **NEXT_PUBLIC_SUPABASE_URL**
   - Значение: ваш Project URL из шага 1
   - Пример: `https://mywoevhpkxjheyikqliz.supabase.co`

   **NEXT_PUBLIC_SUPABASE_ANON_KEY**
   - Значение: ваш anon public ключ из шага 1
   - Должен начинаться с `eyJhbGciOiJIUzI1NiIs...`
   - НЕ используйте ключи в формате `sb_publishable_...` - это неправильный формат!

4. Если переменных нет - создайте их
5. **Сохраните изменения**

### Шаг 3: Пересоберите проект

1. После сохранения переменных Vercel автоматически пересоберет проект
2. Или вручную: **Deployments** → последний деплой → **Redeploy**

### Шаг 4: Проверьте снова

1. Подождите 1-2 минуты после деплоя
2. Откройте `/debug-auth`
3. Нажмите "Проверить подключение к Supabase"
4. Должно быть "✅ Подключение успешно"

## Важно!

- **Формат ключа**: Должен быть JWT токен (начинается с `eyJ`), а НЕ `sb_publishable_...`
- **URL проекта**: Должен соответствовать вашему текущему проекту Supabase
- **После обновления**: Обязательно пересоберите проект на Vercel

## Если проблема сохраняется

1. Проверьте, что проект Supabase активен (не приостановлен)
2. Проверьте настройки CORS в Supabase (обычно настраивается автоматически)
3. Проверьте логи на Vercel в разделе **Deployments** → **Logs**
